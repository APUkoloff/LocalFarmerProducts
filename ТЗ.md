# Требования к веб-приложению Магазин локальных фермерских продуктов
## Общие требования:
1. Интуитивно понятный интерфейс с адаптивным дизайном для всех устройств (десктоп, планшеты, мобильные).
2. Форма заказа с валидацией полей (например, адрес, контакты, способ оплаты).
3. Поддержка браузеров:
     Chrome ≥ 60, Firefox ≥ 55, Edge ≥ 79, Safari ≥ 12.
     Graceful degradation для устаревших браузеров (например, IE11).
4. Безопасность:
    HTTPS, защита от XSS/SQL-инъекций, хэширование паролей.
    Роли пользователей (покупатель, продавец, администратор).
5. Производительность:
    Время загрузки страниц ≤ 2 сек (оптимизация изображений, кэширование).
    Поддержка lazy loading для медиафайлов.
## Функциональные требования:
### Для покупателей:
1. Каталог товаров:
    Фильтрация по категориям (овщи, молочные, фрукты и т.д.).
    Сортировка по цене, названию, рейтингу, дате добавления.
    Показ наличия (шт./кг) с уведомлением при нехватке.
2. Корзина:
    Добавление/удаление товаров, изменение количества.
    Сохранение корзины между сессиями (через cookie/localStorage).
3. Профиль покупателя:
    Редактирование данных (имя, email, телефон, адрес доставки).
    История заказов (архивные, активные).
4. Оформление заказа:
    Выбор способа оплаты (карта, наличные при доставке).
    Уведомления о статусе заказа (email/SMS).
### Для продавцов:
1. Управление товарами:
     Добавление/редактирование/удаление товаров (название, цена, описание, фото).
     Указание количества/веса в наличии.
2. Профиль продавца:
    Редактирование контактов, адреса, реквизитов для выплат.
    Просмотр статистики продаж (графики, выручка).
3. Заказы:
    Просмотр активных и архивных заказов.
    Изменение статуса заказа (например, "В обработке", "Доставляется").
### Для администраторов:
1. Управление пользователями:
    Блокировка/разблокировка аккаунтов.
    Просмотр и редактирование профилей.
2. Аналитика:
    Фильтры по периодам (день/неделя/месяц/год).
    Экспорт данных в CSV/XLSX.
3. Модерация: проверка новых продавцов/товаров.
## Другие функциональные требования (необязательно):
1. Репутация пользователей:
    Рейтинг продавцов (звёзды, отзывы).
    Система жалоб на некачественные товары.
2. Закладки:
    Сохранение любимых продавцов/товаров.
3. Повтор заказов: кнопка "Повторить заказ" для архивных заказов.
4. Интеграции:
    Выгрузка данных для ML-аналитики (например, прогнозирование спроса).
    API для подключения внешних сервисов (например, логистики).
## Технические требования
1. Бэкенд:
    Язык: Python (Django/Flask) или Node.js.
    База данных: MySQL (с резервным копированием).
2. Фронтенд:
  Фреймворк: React/Vue.js.
  UI-кит: Material-UI или Bootstrap.
3. Хостинг:
  Облако (AWS, Google Cloud) или VPS.
  Мониторинг доступности (например, Sentry).
## Нефункциональные требования.
1. Масштабируемость: поддержка до 10 000 пользователей одновременно.
2. SEO: оптимизация мета-тегов, URL-структуры.
3. Локализация: поддержка нескольких языков (например, русский/английский).
